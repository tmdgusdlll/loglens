# 0006. 일일 할당량 초과 시 쿨다운 처리

## 상태
확정됨

## 배경 (Context)
Gemini 무료 티어는 분당/일당 요청 제한이 있다. 할당량을 초과(HTTP 429)했을 때 loglens가 어떻게 동작해야 할지 결정이 필요했다.

## 결정 (Decision)
429 응답을 한 번 받으면 일정 시간(예: 1시간) 동안 AI API 호출 자체를 건너뛰는 쿨다운 상태로 전환한다. 이 기간에도 에러 감지 및 터미널 출력(`[AI 분석 건너뜀 - 일일 한도 초과]`)은 계속되며, 프로그램 전체는 종료되지 않는다.

## 근거 / 검토했던 대안 (Rationale / Alternatives)
- **매번 재시도(단순)**: 구현이 가장 단순하지만, 이미 막힌 걸 알면서도 매 에러마다 헛되이 API를 호출하고, 같은 실패 메시지가 터미널에 반복 출력되어 도배됨.
- **쿨다운 처리(채택)**: 상태 플래그 + 타이머만 추가하면 되는 정도로 구현 부담이 크지 않으면서, 불필요한 API 호출과 터미널 노이즈를 모두 방지.

## 결과 (Consequences)
- `GeminiAnalyzer`는 마지막 429 발생 시각을 기억하는 내부 상태를 가져야 한다.
- 쿨다운 중에 감지된 에러도 `ErrorStore`에는 "분석 건너뜀" 상태로 기록되어 리포트에서 확인 가능하다.
- 재시도 로직(쿨다운 종료 후 자동 재개)은 이 결정에 포함되며 별도 스케줄러 없이 폴링 루프에서 시각 비교로 처리한다.
