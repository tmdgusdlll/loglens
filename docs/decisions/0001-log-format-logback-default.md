# 0001. 로그 포맷: Logback 기본 패턴 고정

## 상태
확정됨

## 배경 (Context)
loglens는 데모 스프링부트 앱이 남기는 로그 파일을 실시간으로 파싱해서 에러(스택트레이스)를 추출해야 한다. 로그 파서를 어떤 포맷까지 지원하게 만들지 결정이 필요했다. 여러 포맷/멀티라인 스타일을 지원하는 범용 파서는 유연하지만 엣지 케이스가 늘어나 초기 구현 복잡도가 커진다.

## 결정 (Decision)
데모 앱의 로그 포맷은 Logback 기본 패턴으로 고정한다. loglens 파서는 이 포맷 하나만 정확히 파싱하도록 구현한다.

## 근거 / 검토했던 대안 (Rationale / Alternatives)
- **범용 파서(정규식 설정 가능)**: 타임스탬프/레벨 패턴을 설정으로 바꿀 수 있어 여러 로그 포맷에 대응 가능. 하지만 멀티라인 스택트레이스 판별 로직까지 일반화하려면 초기 복잡도가 크게 증가.
- **JSON 구조화 로그**: 파싱 자체는 오히려 안정적(정규식 대신 JSON 파싱)이지만, 데모 앱의 로그 설정(logstash-logback-encoder 등)을 별도로 손봐야 하고, 사람이 터미널에서 로그를 직접 읽기엔 불편해짐.
- **Logback 기본 패턴 고정**: 핵심 기능(에러 감지 → 즉시 분석)이 확실히 동작하는 걸 최우선으로 삼기로 했으므로, 파서를 가장 단순하고 견고하게 만들 수 있는 이 방식을 선택.

## 결과 (Consequences)
- 데모 앱의 `logback-spring.xml`은 커스텀 패턴이 아닌 Logback 기본 패턴을 사용하도록 맞춰야 한다.
- 추후 다른 포맷(JSON 로그 등)을 지원하려면 파서를 확장해야 한다(현재 범위 밖).
