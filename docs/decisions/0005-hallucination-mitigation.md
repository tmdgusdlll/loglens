# 0005. 환각(hallucination) 대응 전략

## 상태
확정됨

## 배경 (Context)
AI가 에러 원인을 그럴듯하지만 근거 없이 지어낼 위험(환각)이 있다. 프로젝트 요구사항에서 이를 어떻게 다룰지 명시적으로 고민하도록 요청받았다.

## 결정 (Decision)
다음 네 가지 장치를 함께 적용한다.
1. **소스코드 컨텍스트 첨부**: 스택트레이스가 가리키는 실제 `.java` 파일의 해당 줄 주변(예: 앞뒤 10줄)을 `SourceContextResolver`가 찾아 프롬프트에 포함한다.
2. **구조화된 출력 강제(JSON)**: `{cause, explanation, suggestion, confidence}` 스키마로만 응답하도록 요청해, 모델이 근거가 부족할 때 `confidence`를 스스로 낮게 표시하도록 유도한다.
3. **낮은 temperature(예: 0.2)**: 창작적 응답보다 사실 기반 응답을 유도한다.
4. **출력 라벨링**: 터미널/리포트 어디서든 `[AI 추정 · 확신도: medium]` 형태로 항상 표시해 사용자가 결과를 맹신하지 않도록 한다.

## 근거 / 검토했던 대안 (Rationale / Alternatives)
소스코드 없이 스택트레이스 텍스트만 전달하는 방식도 검토했으나, 모델이 실제 코드를 보지 못한 채 추측하게 되어 환각 위험이 훨씬 커진다. 구현 복잡도가 약간(파일 찾기 로직) 증가하는 대가로 환각을 가장 효과적으로 줄일 수 있는 소스코드 컨텍스트 포함을 채택했다.

## 결과 (Consequences)
- `SourceContextResolver`는 `--watch-dir` 하위에서 스택트레이스의 클래스명에 해당하는 `.java` 파일을 찾아야 한다(패키지 경로 기반 탐색).
- 소스 파일을 찾지 못하는 경우(예: 서드파티 라이브러리 코드) 소스 컨텍스트 없이 분석을 진행하되, 이 경우 confidence가 낮게 나올 가능성이 높다는 것을 프롬프트에 명시한다.
